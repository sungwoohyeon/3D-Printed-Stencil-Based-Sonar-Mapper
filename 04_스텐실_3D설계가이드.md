## 왜 v1(동일길이 라디얼 캡)은 부족한가 (Codex)

내 첫 설계는 20개 홀이 모두 같은 길이(벽 5.5mm)라 **위상이 거의 같은(co-phased)** 점음원 배열 → 다양성이 위치뿐. 40kHz·Ø28mm 개구의 각분해능 ≈ λ/D = 8.6/28 ≈ **18°** → 15cm에서 가로 ~5cm 분해. 20cm 폭을 ~10칸으로 나누기엔 부족. **가변 경로길이가 필수.**

## 왜 360° 원통형인가 (방향 — 검증 완료)

스텐실은 **완전한 360° 원통**이 맞다. (공개 STL 메시 분석: 튜브가 12/12 방위 섹터 균일 분포 / Owlet 코드: 홀 각도 0~360° / Codex 확인.)
- 스텐실은 **지향성 프로젝터가 아니라 근거리 간섭·회절 "코더"** 다. 각 구멍=지연·감쇠 가진 2차 음원, 작은 구멍은 소리를 넓게 회절 → **옆·뒤 구멍도 앞 장면에 기여**. 핵심은 "앞으로 모으기"가 아니라 **구멍 위치·경로길이 다양성**. 근거리(Fraunhofer~58cm≫15cm)라 옆·뒤도 비슷한 거리.
- **Forward-bias(앞쪽 집중)는 비권장:** 앞 SNR은 조금↑ 가능하나, 위치 다양성↓ → H 열 상관↑(조건수 악화) → 복원 품질↓ 가능성.
- **★마운트 방향(논문 Fig.2 확인)★:** **구멍(소리 나오는 면)이 목표를 향한다.** 스피커는 반대쪽 캐비티에 꽂혀 **캡 안으로** 발사(목표 직접 안 봄 — 캡이 코딩), 마이크는 캡 옆. 순서: 스피커→캡(코딩)→구멍에서 코드음→반사→마이크. 논문 헬리컬(`stencil_5cm_mm`)은 구멍이 **끝면**이라 그 면을 목표로 / 우리 v2는 구멍이 **옆면**이라 옆을 목표로(배치 달라 비추천). **H 측정은 이 배치 그대로** ([05](05_벤치_및_실측H_가이드.md)).

## v2 위상캡 설계 (우리 설계, Fusion 빌드·검증 완료)

**원리:** 소리가 바닥 트랜스듀서 → **중앙 Ø10mm 기둥**으로 상승 → 서로 **다른 높이의 16개 라디얼 홀**로 배출. 각 홀까지의 상승 거리가 다르므로 **채널마다 경로길이가 다름**(=Owlet의 위상 코딩 메커니즘).
- Ø30 OD × 40mm. 바닥 Ø16.7 캐비티(16mm 트랜스듀서, 깊이 12mm) + 시팅 레지. 중앙 Ø10 기둥(12→38mm). 측벽 16홀 Ø2.3, 높이 14~37mm.
- **검증(Fusion 실측):** 바디 1개, vol 22.81cm³, 경로길이 **12~35mm(스프레드 23mm ≈ 2.7λ)** → 실질적 위상 다양성. 원통면 20.
- 빌드 스크립트: `models/build_phasecap_v2_fusion.py` (파라미터 수정·재생성용).

## ★레진 MSLA 출력 준비 (Codex 권고 — 필독)★

**방향(orientation):**
- 캡 축을 **30~45° 기울여** 배치.
- **트랜스듀서 캐비티(개방부)는 아래(베드/배트 쪽)로** 향하게 → 밀폐 진공(suction) 방지.
- 모든 채널·캐비티가 양끝 개방이면 **드레인 홀 불필요**. (공개 STL 나선 튜브가 막힌 곳 있으면 출력 후 IPA 주사기로 관통.)

**서포트:**
- **외부에만** 서포트. 캐비티(Ø16.7)·기둥·홀 **내부엔 절대 금지**.
- 출구 림 근처는 가벼운 서포트 가능.

**홀 막힘 방지:**
- Ø2.3mm로 모델링됨(목표 Ø2mm + 여유). 35~50µm MSLA에서 출력 OK.
- **출력 직후·경화 전** 모든 채널에 IPA 주사 → 에어블로어로 관통 확인. **경화 후엔 못 뚫음.**

**트랜스듀서 장착:**
- 16mm 트랜스듀서 ↔ Ø16.7 캐비티 = **슬립 핏**(억지 끼움 금지, 레진 깨짐). 헐거우면 정상.
- 삽입부 0.5mm 모따기(없으면 손으로 살짝 다듬기). v2엔 시팅 레지 있음.
- **둘레를 RTV 실리콘/왁스로 기밀 밀봉** — 누설이 성능 최대의 적.
