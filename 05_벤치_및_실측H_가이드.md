# 벤치 + 실측 H 가이드

복원의 성패는 **H를 얼마나 정확히 측정하느냐**에 달렸다. 수식 H는 시작점일 뿐, 사용 가능한 H는 측정해야 한다(G0가 증명).

## 1. 회로

```
ESP32-S3 GPIO4 ──> [SN74LVC2G04 버퍼] ──> 40kHz 스피커(송신)
                                              )))  스텐실
스피커  ──흡음 배플──  마이크   (직접 누설 차단!)
40kHz 마이크 ──> [OPA344 ~40dB + 밴드패스 35~45kHz] ──+1.65V 바이어스──> ESP32-S3 ADC(GPIO?)
ESP32-S3 ──USB──> PC
```
- **송신:** ESP32 PWM(40kHz)은 약하므로 SN74LVC2G04로 버퍼(또는 트랜지스터). 5V 구동이면 출력↑.
- **수신 바이어스:** ADC는 0~3.3V만 → 프리앰프 출력에 +1.65V DC 더해 중앙 정렬(저항분압 + 1µF 결합).
- **밴드패스:** 35~45kHz만 통과(가청 잡음·DC 제거). 간단히는 RC 1차 ×2.
- **★배플★:** 스피커와 마이크 사이 흡음폼/칸막이. **직접 누설(crosstalk)이 근거리 최대의 적.**
- **★마운트 방향(논문 Fig.2 확인):** **구멍(소리 나오는 면)이 목표를 향한다.** 스피커는 반대쪽 **캐비티에 꽂혀 캡 안으로** 발사(목표를 직접 보지 않음, 캡이 코딩). 마이크는 캡 옆에서 반사음 청취. 순서: 스피커→캡(코딩)→구멍에서 코드음→목표 반사→마이크.
  - 논문 헬리컬 스텐실(`stencil_5cm_mm`): 구멍이 **끝면**에 있음 → 그 면을 목표로, 스피커는 뒤 캐비티.
  - 우리 v2(옆구멍): 구멍이 **옆면** → 옆면을 목표로 (배치가 달라 비추천).
  - **이 배치(구멍 면·스피커·마이크 위치)를 고정한 채 H 측정**, 이후 측정도 같은 배치 유지(바뀌면 H 무효). 왜 360°인지는 [04 §왜 360°](04_스텐실_3D설계가이드.md).

## 2. 펌웨어 업로드 (사용자는 업로드만)

1. Arduino IDE → 보드 매니저에서 **esp32 core 3.x** 설치(`analogContinuous` 필요).
2. 보드 = "ESP32S3 Dev Module", 포트 선택.
3. `bench/esp32_firmware/spidr_bench.ino` 열기 → 상단 상수 확인:
   - `TX_PIN`, `RX_ADC_CH`(보드 핀맵 확인), `SAMPLE_HZ=200000`, `WIN_SAMPLES=240`.
4. 업로드 → 시리얼 모니터(2,000,000 baud)에 `# SPiDR bench ready` 뜨면 OK.

## 3. G1 — 전자 검증 (스텐실 없이)

1. **Q/링다운 측정:** 버스트 1발 송출 → 마이크 파형 확인. 감쇠시간 ≈ Q/(π·40k). 6cm 왕복(0.35ms)보다 길면 근거리 가림 → 배플·게이팅 강화.
2. **직접 누설:** 물체 없이 캡처 → 송신 직후 큰 누설이 보임. 이게 **배경(blank)**.
3. **단일 반사:** 6/10/15cm에 판 1개 → 배경 빼면 에코가 시간축 다른 위치에 보여야 함(ToF). 안 보이면 게인·배플·게이트 조정.
   - ⚠️ ESP32 ADC가 200kHz 연속을 못 버티면(드롭/지터) → `SAMPLE_HZ` 낮추거나 라즈베리파이 Pico 폴백([02](02_부품_구매가이드.md)).

## 4. G2/G3 — 실측 H (코어)

스텐실 장착 후:

```bash
cd bench
python measure_H.py --sim                 # (먼저) 하드웨어 없이 파이프라인 점검 → H_measured.npz
python measure_H.py --port COM5 --pitch 2 --depth 10   # 실제 측정 (포트는 장치관리자 확인)
```

**프로토콜(measure_H.py가 안내):**
1. **배경:** 물체 다 치우고 측정 → blank.
2. **스캔:** 작은 점반사체(Ø6mm 금속구/볼트)를 픽셀 격자 각 위치로 이동하며 측정.
   - 1D 먼저: 가로 −10~+10cm, 2cm 간격(11점), 깊이 10cm 고정. **손+자+3D프린트 지그로 충분**(모터 불필요).
3. 각 측정 = (blank 차감) + 동기 평균(기본 32회, SNR↑) → H의 그 열.
4. 완료 → `H_measured.npz`. 출력되는 **열 결맞음·조건수**로 코드 품질 확인(낮을수록 좋음).

**음속/온도:** `c = 331.3 + 0.606·T(°C)`. 1°C 변동도 수십° 위상 이동 → 측정 중 온도 기록, 필요시 보정.

## 5. 실제 장면 복원 (G3)

```python
import numpy as np, sys; sys.path.insert(0,'../sim')
from reconstruct import reconstruct_nnls, detect_recall
d = np.load('H_measured.npz'); H = d['H']
# 실제 장면(물체 배치) 1회 캡처 → y (blank 차감, to_column)
# x_hat = reconstruct_nnls(H, y)  → 점유맵
```
- 시작은 단일 물체 위치 맞히기 → 두 물체 2cm 분해 → 거친 2D(2cm 격자, depth 여러 층 측정).
- **SNR 부족하면 평균 횟수↑**(논문 12fps는 임베디드 최적화 결과 — 벤치 초기엔 느려도 정상).

## 6. 트러블슈팅

| 증상 | 원인/조치 |
|------|-----------|
| 에코 안 보임 | 누설·링다운이 가림 → 배플 강화, 레인지게이트 시작 늦춤, 게인↑ |
| 복원 잡음 많음 | H 결맞음 높음 → 스텐실 재설계(홀 다양성↑) 또는 주파수/평균↑ |
| 위치 어긋남 | 마이크/픽셀 좌표 오차(수mm=큰 위상) → 좌표 정밀 재측정 |
| ADC 드롭 | 200kHz 연속 실패 → SAMPLE_HZ↓ 또는 Pico |
| 클리핑 | 프리앰프 게인↓ 또는 바이어스 재조정 |
