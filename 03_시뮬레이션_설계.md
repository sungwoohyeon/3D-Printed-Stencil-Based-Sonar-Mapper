# 시뮬레이션 설계 (G0)

## 목적과 위치

시뮬레이션은 **이 재현의 본체가 아니다**(본체는 실측 H — [05](05_벤치_및_실측H_가이드.md)). 시뮬의 역할:
1. **복원 알고리즘·접근법 검증** (하드웨어 전, 무위험)
2. **스텐실 설계**(튜브 길이) 산출 → 3D 모델 파라미터 ([04](04_스텐실_3D설계가이드.md))
3. **"모델만으로는 부족함"을 정량 증명** → 실측 H 필요성의 근거

## 핵심 설계 결정 (Codex 리뷰 반영)

| 항목 | 잘못된 단순화 | 채택한 설계 |
|------|--------------|------------|
| H 종류 | 단일주파수 위상 H | **시간영역 시계열 H** (톤버스트·ToF) |
| 신호 경로 | 스텐실→픽셀 (한쪽) | **왕복** 스텐실→픽셀→마이크, 진폭 1/(r_np·r_pm) |
| 측정 차원 M | 5주파수만 | (게이트 시간샘플 240) × (5주파수) = **1200** |
| 검증 | 같은 H 자기복원 | **모델 불일치**(섭동 물리)로만 |
| 거리 가정 | 원거리 | **근거리 구면 전파** + 마이크 좌표 |

## 모듈 구조

- **`forward_model.py`** — `Config`(음속·fs·주파수·스텐실 기하·레인지게이트), `tone_burst`(Hann 톤버스트), `make_stencil`(튜브 출구·길이), `scene_grid`(픽셀 격자), `received_signal`(픽셀 1개의 왕복 수신 시계열, 튜브에 벡터화), `build_H`(M×N).
- **`reconstruct.py`** — `reconstruct_nnls`(비음 최소제곱, 기본), `reconstruct_tikhonov`(LSQR), `reconstruct_lasso`(sklearn). 열 정규화로 조건수 개선. 평가: `detect_recall`(상위 k 피크가 참 타깃 잡은 비율), `image_corr`.
- **`design_stencil.py`** — Owlet식 랜덤서치로 튜브 위치·길이를 최적화(열 결맞음 최소화). `stencil_design.npz` 저장 → 3D 모델이 사용.
- **`validate_mismatch.py`** — G0 본체. 섭동 물리로 데이터 생성 → 공칭 H로 복원 → 불일치 수준별 정확도 스윕.

## 실행

```bash
cd sim
python forward_model.py        # H 형상·결맞음 확인
python design_stencil.py       # 스텐실 최적화 → stencil_design.npz
python validate_mismatch.py    # G0 게이트
```

## G0 결과 (실측 출력)

스텐실 최적화: 최대 결맞음 0.74(무작위) → **0.54(최적)**, 조건수 12.8. 튜브 20개, H=(1200, 11), 2cm 픽셀, SNR 30dB.

**모델 불일치 스윕 — 단일/이중 타깃 모두:**

| 불일치 수준 | 타깃검출 recall (단일) | (이중 2cm) |
|------------|:---:|:---:|
| 잡음만 | 100% | 100% |
| 작음(보정됨, ~10°) | **100%** | **100%** |
| 중간(~30°) | 90% | 88% |
| 큼(미보정, 180°) | 47% | 53% |
| 인버스크라임(참고) | 100% | 100% (무의미) |

**→ G0 게이트 PASS.** 해석:
- 튜브 전달을 잘 보정한(작은 불일치) 경우, 복원기가 **2cm 간격 두 물체까지 100% 분해** → 알고리즘·접근법 타당.
- 불일치가 커지면(튜브 음향 미보정) **정확도 붕괴(47~53%)** → **수식 H만으로는 안 되고 실측 H가 필수**임을 수치로 증명.
- 인버스크라임(같은 H 자기복원)은 항상 100% → 이것만 보면 속는다(그래서 불일치 검증을 한 것).
