## 2. 해결 문제

곤충 크기(수 인치) 초소형 로봇은 **SWaP-C**(Size·Weight·Power·Cost) 제약이 극심하다. 로코모션에 전력을 쓰고 나면 센싱에 **10~30mW**만 남는다. 기존 depth 센싱은 부적합:
- **LiDAR:** 기계적 스캔 → 전력·크기 과다
- **빔포밍 소나/위상배열:** 다수 트랜스듀서 + 병렬 송수신 체인 → 연산·전력 과다

SPiDR은 **단일 스피커/마이크 + 패시브 구조물**로 이 문제를 우회한다.

## 3. 핵심 아이디어 — 구조-보조 공간 코딩

단일 무지향성 스피커는 모든 방향에 같은 신호를 보내 공간 정보를 못 만든다. SPiDR은 스피커 위에 **3D프린팅 "스텐실"**(다공성 캡)을 씌운다:
- 내부에 **길이가 다른 여러 튜브(나선 경로)**가 있고, 각 튜브 출구는 다른 방향을 향한다.
- 각 출구는 **고유한 지연·감쇠(복소 게인)**를 가진 **이차 음원**처럼 동작한다.
- 이 이차 음원들이 간섭하여, **공간의 각 지점마다 고유한 신호 패턴(코드)**을 투사한다.
- 멀티패스(다중경로)를 통신에선 잡음으로 보지만, 여기선 **의도적으로 설계**해 공간 코드로 쓴다.

> 연구 계보: **Owlet**(MobiSys'21, 마이크측 메타물질로 방향 추정 DoA) → **Sirius**(MobiSys'23, RF 자기위치) → **SPiDR**(깊이 이미징).

## 4. 측정 모델 — `y = H·x` (★시간영역 ToF★)

장면을 거리 평면 위 픽셀들로 이산화한다(20cm폭 × 10cm깊이, 1cm → N=200 픽셀).
- **`H`** : M×N 채널 행렬. 각 열 = "픽셀 i에 이상적 반사체가 있을 때 마이크가 받는 신호".
- **`x`** : 픽셀 점유 벡터(반사율). 물체 있으면 큰 값, 없으면 0.
- **`y`** : 마이크 수신 신호. `y = Hx` → `x`를 역으로 복원.

> **⚠️ 재현 시 가장 중요한 정정(Codex 리뷰):** `y`는 단일 주파수 위상값이 아니라 **시간영역 신호**다. 논문은 **10주기 톤버스트**(38–42kHz, 1kHz 간격 5개 주파수)를 쏘고 **비행시간(ToF)**으로 거리를 안다.
> - **M = (레인지게이트 내 시간 샘플 수) × (주파수 수)**. 논문 예: 5ms @ 200kHz = 1000샘플 → M≈1000.
> - 따라서 `H`의 각 열은 단일 복소수가 아니라 **샘플된 시계열(임펄스응답)**이어야 한다. CW 위상 모델만 쓰면 거리 차원(=측정의 대부분)을 잃는다.
> - 신호 경로는 **왕복**: 스피커→스텐실 출구→픽셀→반사→마이크. 진폭은 `1/(r_송신·r_수신)`(단일 `1/d` 아님).

## 5. 4대 핵심 기법

### 5.1 멀티패스 합성 (스텐실 최적화)
임의 스텐실은 충분한 코드 다양성을 못 준다. **각 픽셀에서의 신호 상관도를 낮추도록** 튜브 길이를 최적화한다. 논문은 "홀로그래피의 Weighted Gerchberg-Saxton(GS) 알고리즘과 유사"라고 기술. (실제 공개된 Owlet 코드는 **랜덤서치** — 6절 참조.)

### 5.2 주파수 스태킹
튜브 길이가 같아도 **주파수가 바뀌면 위상 코드가 달라진다.** 38–42kHz의 5개 주파수 측정을 결합(스태킹)해 H의 행 수(측정 수)를 늘려 복원 품질을 높인다.

### 5.3 모션 스태킹
로봇 이동 중 같은 장면의 여러 스냅샷을 **mean/median 스태킹**으로 결합(O(n)/O(log n))해 잡음·허상 억제. 최대 170fps 가능.

### 5.4 Fractional Scene Computing (저전력 복원)
전체 H 역변환 대신: ① 인접 상관 픽셀을 **메타픽셀**로 묶어 축소 H로 거친 복원 → ② 물체 있는 영역만 **반복적 고해상도 정밀화** → ③ 역행렬을 **사전계산·프리페치**. Raspberry Pi 4에서 최적화 조합 시 **1380× 에너지 절감**.
> 이 기법은 **임베디드 저전력**용. 데스크탑 PC 재현에선 불필요(표준 역변환으로 충분).

## 6. 포워드 모델 수식

논문 eq.(1) — 픽셀 m에서의 음압:

```
p_m = i·(ρ c k S / 2π) · Σ_{n=1..N} a_n · e^(i2πft) · e^(−i k d_mn) / d_mn
```
- `a_n, ` = 이차음원 n의 진폭·시간지연, `f` = 주파수, `k` = 2πf/c (파수)
- `d_mn` = 음원 n→픽셀 m 거리, `ρ, c` = 공기 밀도·음속, `S` = 반사면적(단위상수)
- Weighted GS(홀로그램, Di Leonardo 2007)와 동형.

**Owlet 공개 코드(`ref_optimizeStencil.m`)의 실제 모델** — 마이크측이지만 물리 동일:
- 실린더(내경 17.5 / 외경 30 / 높이 60 mm), 홀 20개를 (높이, 각도)로 배치
- 홀 기여: `(D/(jλr²))·exp(−j·2π/λ·(r + added_distance))·cardioid(angle)`
  - `r = √(D² + 홀높이²)`, `added_distance` = 외벽을 감아 도는 추가 경로(= 위상 코드 원천)
- 최적화 = **랜덤 홀패턴 N개 생성 → 각도별 주파수 시그니처의 다양성(인접 각도 차이 합) 최대 → 최선 선택.** (gradient WGS 아님)
- 측정 지향성(`cardoid_used.mat`)으로 스피커/마이크 방사패턴 보정.

→ 재현 시 이 코드를 **스피커측(송신)으로 변형**하고 `added_distance`를 **제어 가능한 튜브 길이 `l_n`**으로 바꾸면 된다.

## 9. 한계 (논문 명시 + 재현 관점)

- 감지 범위 6~15cm로 제한(멀수록 반사 약화 → 블러). **근거리(near-field)** 영역(Fraunhofer 거리 ≈0.58m ≫ 15cm)이라 원거리 근사 불가.
- 희소 장면(sparsity 60%+) 특화, 밀집 환경 성능 저하.
- 2D 단면만(완전 3D는 미래 과제).
- 40kHz 한계로 해상도 제약.

## 주의사항 · 불확실성

- **시간영역 측정모델**을 놓치면 거리 정보를 잃는다(4절). 재현 코드는 톤버스트·ToF·게이팅을 반영해야 한다.
- **고Q 피에조 링다운**(~0.24ms)이 6cm 왕복(0.35ms)과 겹쳐 근거리 에코를 가릴 수 있다 → Tx/Rx 분리·배경차감 필수([05](05_벤치_및_실측H_가이드.md)).
- 스텐실 STL은 **미터 단위** → 출력 시 ×1000.
- 논문 SSIM>80%는 **실측 H + 캘리브레이션**이 전제. 데스크탑 1차 재현의 정직한 목표는 1D/거친 2D.
