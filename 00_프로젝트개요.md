# SPiDR 재현 — 프로젝트 개요

## 한 줄 정의

**40kHz 스피커 1개 + 마이크 1개 + 3D프린팅 패시브 "스텐실" 1개**로, 물체에 반사된 초음파를 분석해 책상 위 **2D 단면 점유맵(depth-map)**을 만드는 시스템을 직접 재현한다.

```
   [ESP32-S3]──PWM 40kHz──>[스피커]──))) 
        │                      └─[스텐실: 내부 튜브가 방향별 고유 코드 생성]
        │                              )))  ──> [물체] ──반사──┐
        └──ADC<──[프리앰프]<──[마이크]<───────────────────────┘
        │
        └──USB시리얼──> [PC: y=Hx 복원 → 2D 맵]
```

## 핵심 원리 (왜 카메라 없이 2D가 나오나)

스텐실 내부의 길이가 다른 여러 튜브가 하나의 스피커 소리를 **방향마다 다른 진폭·위상 조합(공간 코드)**으로 쪼개 내보낸다. 어떤 위치에 물체가 있으면 그 위치 고유의 코드가 마이크로 되돌아온다. 모든 위치의 코드를 미리 아는 행렬 `H`에 담아두면, 받은 신호 `y`에서 물체 위치 `x`를 역으로 푼다: **`y = H·x`**.

> **이 프로젝트의 핵심 통찰(Codex 리뷰):** `H`를 수식으로 "계산"하려 하면 실제와 안 맞는다(근거리·튜브 음향·왕복 경로). 대신 **점반사체를 격자 위로 옮기며 `H`를 직접 "측정"**하면 부정확한 모델 없이도 영상이 나온다. → **실측 H가 이 재현의 심장.**

## 성공 기준 (게이트형 단계)

| Gate | 목표 | 통과 기준 |
|------|------|-----------|
| **G0** 시뮬레이션 | 복원 알고리즘 검증 | 섭동 모델로 만든 데이터를 다른 H로 복원해 단일 타깃 위치 맞히기 (자기복원❌) |
| **G1** 전자 | ESP32 40kHz 송수신 | 6/10/15cm 점반사 에코를 배경차감 후 분리 검출 |
| **G2** 스텐실+1D | 스텐실 출력 + 1D 실측 H | 두 물체를 좌우로 분해 |
| **G3** coarse 2D | 실측 H로 2D 복원 | 2cm 격자 점유맵 생성 |

## 정직한 기대치

- **현실적 1차 목표:** 1D 또는 거친(2cm 격자) 2D 점유맵. 성공 확신 ~50~60%.
- **Stretch goal:** 논문 수준 1cm 격자·2cm 간격 분해·SSIM>80%. 확신 ~20~30%.
- 논문의 9.9mW·12fps·CR2032 3일은 **임베디드 저전력 최적화**라 데스크탑 재현 범위 밖(후속).
